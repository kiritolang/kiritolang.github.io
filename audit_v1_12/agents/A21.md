## How to read the map

- **Coverage rating** per symbol group: **WELL** = happy-path + at least some adversarial/edge; **THIN** =
  smoke/happy-path only, edge/error paths missing; **UNTESTED** = no direct test found.
- A symbol being documented but returning 0 precise `.method(` hits in the corpus was verified against
  CLAUDE.md/docs to be a *real* Kirito symbol before flagging (several Python-ish names like
  `casefold`/`swapcase`/`title`/`splitlines`/`isnumeric`/`difference_update` are **not** part of
  Kirito's surface — their 0-hits are correct absences, not gaps).

## Prioritized gap list (the harden-phase work queue)

### HIGH — untested surface reachable with hostile input / crash surface

1. **Native-HOF recursion → SIGSEGV (A04-1).** No test drives recursion through `sorted(key=g)`,
   `xs.sort(key=g)`, `min/max(key=g)`, `container.apply(g)` where `g` recurses. Existing depth-guard
   tests only do shallow Kirito→Kirito recursion. `sorted` is a builtin → affects **embedders**. Add a
   `.ki` (and C++) test asserting a *catchable* error, not a crash, at the release stack size.
2. **GC-rooting of `iterate()`/snapshot fresh handles (A06-1/2, A09-3/4).** Whole class untested:
   `Set()`/`Dict()` constructors, `zip`/`map`/`filter`/`sorted`/`enumerate`/`all`/`any`, and
   `List/Dict/Set.apply`/`List.sort` — with a **non-interned** fresh-handle source and an allocating
   callback under `setGcThreshold(1)`. Must survive without a dangling-handle throw. (Test authors:
   avoid bare literal elements — they are pinned bytecode consts and mask the bug.)
3. **`pow(base, exp, mod)` large-modulus overflow (A09-1).** Every test uses a tiny mod. Add
   `pow(2**62, 2, INT64_MAX)`-class cases asserting the exact residue (guards the int64 pre-widen UB).
4. **`InstanceValue::equals` native-Instance downcast (A07-1).** A `.ki` storing a native
   `ValueKind::Instance` (DateTime/Bytes/Matrix) in a Dict/Set alongside a user class with
   `_eq_`/`_hash_`, colliding-hash — run under the ASan/UBSan binary.
5. **pool-leak-per-`spawn` (A08-1).** Sanitizer-invisible (pool bypassed under asan). Needs a
   release-mode RSS/leak assertion over many `parallel.spawn` iterations.
6. **`replace`/`join` output-size guard (A06-8).** Add `errors/*.experr`:
   `("a"*100000).replace("a","b"*3000)` and `("x"*3000000).join([""]*200)` must **throw**, not build a
   >256 MB String (parity with the existing `repeat_too_large`/`range_too_large`).

## Summary

- **Public surface enumerated (rough):** ~40 builtins + 8 type constructors; ~90 built-in-type methods
  across String/Bytes/List/Set/Dict/Integer/Float; ~30 stdlib modules exposing **~400** functions/
  methods (tensor ~150 alone); ~50 C++ embedding-API surface points. **~550–600 public symbols total.**
- **Coverage verdict:** the corpus is **broad and generally deep** — nearly every module has a base +
  `_deep` unit file plus `spec_*.ki` golden scripts, and precise-grep confirms essentially every
  *documented* symbol is referenced somewhere. **Very few symbols are UNTESTED outright** (the
  clearest: `.ki`-level lexer token tests, `asList`/`callKw` in the C++ API, and several documented-quirk
  lock tests).
- **The dominant gap is depth, not breadth:** ~**60–80 symbols/features are THIN** — happy-path only,
  missing the adversarial/edge/error-path cases that the v1.12 goal ("not a single method, attribute,
  class, argument or feature untested… from every angle") demands.
- **Top areas most in need of tests (in order):**
  1. **GC-rooting under callback-driven mutation** — the single highest-leverage class (A04-1, A06-1/2,
     A09-3/4): native-HOF recursion crash + fresh-`iterate()`-handle dangling across ~10 builtins/methods.
  2. **Lexer unit + parser-boundary tests** — the front-end is under-unit-tested (skeletal
     `test_lexer.cpp`), relying almost entirely on end-to-end coverage; token-level + depth-guard-boundary
     + diagnostic-text assertions are missing.
  3. **Special-method / kwarg-binder error paths** — `makeMethod` omitted-required-arg, dunder
     return-type violations, reflected-operator asymmetry.
  4. **Output-size & numeric-overflow guards** — `replace`/`join` OOM, `pow(...,mod)` large modulus,
     Set/Dict churn, slice/`**` INT64 edges.
