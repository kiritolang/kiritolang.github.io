# v1.16.1 — full doc-vs-impl coverage campaign (LABEL X loop)

**Goal (user-directed):** page-by-page, feature-by-feature, compare every documented function / class /
method / attribute in Kirito's builtins + standard library against the real implementation; verify the
impl is correct; add anything missing from the docs; and write comprehensive, dedicated `.ki` tests —
normal use + self-determined **edge cases** + **hardening** (does bad input raise the *correct* error
message? does a case that *should* error actually error, or is it a **silent error**?). Cover BOTH
functioning and error paths. Do it thoroughly even where prior tests exist.

**Constraint:** do NOT modify `src/` (no C++) — only `.ki` + `.expected` — UNLESS a real bug is found,
which is then a separate, gated fix. Each new test file: `tests/scripts/cov_<area>.ki` (+ `.expected`
generated by the actual `build-release/ki` / `build-debug/ki`). Validate with
`tools/scripts/bytecode_stability.sh` (must stay 0-diverged) after each batch.

**LABEL X** = "recompute what's still uncovered from this table and dispatch the next batch." Loop back
here until every row is `done` and the doc-vs-impl deltas are all resolved.
