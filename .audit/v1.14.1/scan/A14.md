## A14-1 — statistics.quantiles clamps endpoints instead of extrapolating (diverges from Python exclusive method)
- **file:line**: stdlib_kimodules.hpp:455-476 (`quantiles`)
- **severity**: medium
- **scenario**: The "exclusive" method (the documented default) must EXTRAPOLATE when a cut point falls outside the interior interval `[1, ld-1]`; the code CLAMPS to the min/max element instead.
  - `quantiles([10, 20])` → Kirito `[10.0, 15.0, 20.0]`, Python `statistics.quantiles([10,20])` → `[7.5, 15.0, 22.5]`.
  - `quantiles([1,2,3,4,5], 10)` → Kirito `[1.0, 1.2, ... 5.0]`, Python → `[0.6, 1.2, ... 5.4]` (first/last cut points differ).
- **root cause**: the `if lo < 1: append s[0]` / `elif lo >= ld: append s[ld-1]` clamps rather than linearly extrapolating with a negative/over-`n` `delta` the way CPython's exclusive method does (`interpolated = (data[j-1]*(n-delta) + data[j]*delta)/n` with `j` clamped to `1..ld-1` but `delta = i*m - j*n` allowed to go out of `[0,n]`).
- **triggerability**: only when a requested cut point lies outside the interior — i.e. small datasets or `n` large relative to `len(data)`. Interior quantiles (the common case) already match.
- **proposed fix**: mirror CPython exactly: `m = ld + 1`; for each `i` compute `j = clamp(i*m // n, 1, ld-1)`, `delta = i*m - j*n`, `out.append((s[j-1]*(n-delta) + s[j]*delta) / n)`. This keeps interior results identical and fixes the extremes.
- **coverage gap**: existing tests presumably only exercise interior cut points; add a small-dataset case (`[10,20]`) to pin the extrapolation.
