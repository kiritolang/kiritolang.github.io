# A20 — Test-coverage parity audit ("nothing untested")

Round: v1.15.1. Scope: **the test suite itself as an artifact** — its structure, blind spots, and
C++ vs `.ki` parity. Not per-function bug hunting (that's A01–A19).

Mandate audited against (user, verbatim):
> "not a single method, attribute, class, argument or feature shall go untested... Every class,
> method, attribute, function in kirito's builtin and standard library must be tested from every
> possible angle just as C++ reference. Check if c++ tests implemented cover what's written above.
> If not, extend tests. Then do the same for .ki tests."

Binary: `./build-debug/ki` → `ki (Kirito) 1.15.0`. **No builds run** (README hard rule).

Suite size at start: 132 `tools/tests/unit/*.cpp`, 371 `tools/tests/scripts/*.ki`,
303 `tools/tests/errors/*.ki`.

## Method

1. Generate the public-surface inventory mechanically from the live binary via `inspect()` —
   not by reading CLAUDE.md and guessing.
2. Inventory the tests by file/convention.
3. Cross the two per-name, with 4 axes: C++ test / `.ki` test / error-path test / edge-case test.
4. Report systematic blind spots (shape-of-the-problem, not name-by-name).

**Grep-presence is not coverage.** A prior round claimed "all 206 public names have ≥3 test
references" by grepping. A name appearing in a test file proves nothing about its arguments,
its error paths, or its edge values. Findings below distinguish *mentioned* from *probed*.

### A20-0: the previous round's inventory was 22% of the real surface  [severity: HIGH] [confidence: confirmed]

- What: a prior round's headline claim was **"all 206 public names have ≥3 test references"**. The
  live binary exposes **945**. So the "all public names" the round certified were **22% of the
  surface**; the other 739 names were not merely under-tested, they were *not on the list*. This is
  the same failure shape as v1.15's write-barrier soak: the instrument was trusted without checking
  what it could see.
- Evidence: `wc -l A20_surface_flat.txt` → 945, generated mechanically from `inspect()`.
  The 206 figure most plausibly counted builtins + module-level functions of the *native* modules
  only — it misses all 273 native-class members (`Socket.*`, `Tensor.*`, `Regex.*`, `Match.*`,
  `DateTime.*`, `BigInt.*`, `Queue.*`…), all 151 Kirito-authored class members (`DataFrame.*`,
  `Series.*`, `Counter.*`, `Element.*`, `Parser.*`…), and the 77 built-in type methods.
- Impact: every downstream "coverage is complete" statement inherits the undercount.
- Fix: the inventory must be **generated**, never curated. `A20_surface_flat.txt` + the enum scripts
  should be checked in (proposal T1 below) so the surface list can never silently drift again.

### A20-0b: `inspect()` itself under-reports optionality, so it cannot be the coverage oracle alone

Several natives print their params with **no default marker** even though the argument is optional:

```
String value:
  find(sub, start, end) -> Integer          # start/end ARE optional
  replace(old, new, count) -> String        # count IS optional
  split(sep, maxsplit) -> List              # both optional
Regex object:
  findall(string, pos, endpos) -> List      # pos/endpos optional
Socket object:
  recv(n) -> Bytes                          # n optional
  shutdown(how)                             # how optional
Match object:
  group(index) -> String                    # index optional
```

Contrast a signatured native, which does show them: `math.log(x: Number, base = None)`,
`Tensor.compare(other, rel_tol = 1e-09, abs_tol = 0.0)`. So the surface splits into
**signatured** natives (keyword-callable, `inspect`-complete) and **hand-rolled arity-checking**
natives (positional-only, optionality invisible). That split is exactly the A19-1 fault line —
"every `.compare()` that omits a default" — and `inspect` cannot tell you which methods have an
omittable argument. **Coverage for the second group has to be written by hand, per method.**
